Phase 0: Requirements Specification

This plan and focus of this website is to create an app that can convert a given weight into gold. This will be done by using a database to store the conversions, recieving input given by the user, using an api from nasdaq, and creating our own api to handle the conversions.

Phase 1: Design

The first step is to build the django server and html side of the website. Creating a quick template and input boxes for the website is simple enough

From there I'll focus on recieving the data from nasdaq and implementing that into my website

I will then get the database up and migrated and then I'll focus on the unitconv app. The unitconv app will probably take the most time as that is the api and that will be something new and unfamiliar. The idea behind the unitconv will be to create an api that can take an input of a unit and change it to a different unit

Phase 2: Implementation

Created Django server and html for the site. Using an input box for numbers and a selection box for the different units.

Using the fetch api I can get the dataset from nasdaq. Organize the data and display it to the website

Create Conversion database with unit name and conversation rate. I decided to have my base conversion be in tons as it makes the math simpler (Multiply to convert from tons, divide to convert to tons)

Create unitconv api that has proper error handling. Will take from, to, and value. Using these it will convert the value from the from unit to to.

I then fetch the data when the user presses the button and display it in a div. This div if clicked on will be deleted. Handle errors here and return a different message.

Phase 3: Testing & Debugging

Tested no from, to, and/or value in unitconv api. Tested strings instead of integers in unitconv api. Tested negative values in unitconv. Tested string in number input for gold app. Tested changing option in select. Tested not inputing anything in number input. Tested negative numbers. Tested blocking nasdaq and unitconv.

After all the testing no major bugs have been found.

Phase 4: Deployment

Cloned and tested in a new location. Everything ran smoothly
